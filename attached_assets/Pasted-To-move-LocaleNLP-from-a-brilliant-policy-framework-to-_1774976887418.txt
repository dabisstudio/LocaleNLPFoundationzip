### 1️⃣ THE TECHNICAL BLUEPRINT: API Gateway & Monetization Engine

You cannot manually email CSV files to researchers or enterprises. LocaleNLP must operate via a programmable, metered API. This architecture ensures that Academics get free access, Enterprises get billed accurately, and scraping bots are blocked entirely.

**The Recommended Tech Stack:**
* **Authentication & Database:** Supabase (Open-source Firebase alternative, highly respected by developers).
* **API Gateway:** Cloudflare API Shield or Kong (Handles rate limiting and security at the edge).
* **Billing Engine:** Stripe Metered Billing (Connects API usage directly to enterprise credit cards/invoicing).

**The 3-Tier Execution Flow:**

**Tier 1: The Academic Gateway (Zero-Cost, High Verification)**
* **The Flow:** A researcher signs up at `platform.localenlp.org`.
* **Verification Logic:** The system automatically checks if the email ends in `.edu`, `.ac.uk`, or a known NGO domain. If verified, Supabase generates a standard `API_KEY`.
* **Rate Limiting:** The API Gateway restricts this key to a generous but finite limit (e.g., 50,000 requests/month) to prevent a university project from accidentally running up your server costs.
* **The UX:** They get a beautiful, frictionless dashboard showing their usage and a link to the citation guidelines for their published papers.

**Tier 2: The Enterprise Gateway (High-Volume, Metered)**
* **The Flow:** An enterprise (e.g., a telecommunications company building a Swahili chatbot) requests access. Once approved, they input their billing details via Stripe.
* **The Tech:** Supabase issues a high-throughput `ENTERPRISE_API_KEY`.
* **The Metering:** Every time their servers ping the LocaleNLP API for data, the API Gateway logs the request size (in megabytes or tokens). At the end of the month, a webhook triggers Stripe to automatically calculate the usage and issue an invoice (e.g., $0.002 per token).

**Tier 3: The Contributor Payout Routing (The Flywheel)**
* **The Tech:** As Stripe collects the Enterprise revenue, an automated script runs at the end of the month. It queries the database to see which Lughatna contributors had their validated data accessed.
* **The Payout:** Using an API like Flutterwave or Paystack, the system routes a percentage of that revenue directly to the contributors' mobile money wallets (M-Pesa, Wave, Orange Money).

**Why this wins:** You have completely automated the Sovereign IP model. It requires zero manual accounting, scales infinitely, and proves to auditors that the money is flowing exactly as promised.
